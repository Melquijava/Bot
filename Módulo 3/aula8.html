AULA 8 • Módulo 3 – Dados & Persistência

Mini Banco de Dados com JSON

Objetivo da aula: entender o que é persistência de dados, como usar arquivos JSON como um “mini banco de dados” e como isso se conecta com sistemas de XP, economia, configurações e estatísticas do bot.

▸ 1. O que é persistência de dados?

Até agora, tudo o que o bot faz “some” quando ele é desligado. Persistência de dados é a capacidade de guardar informações para usar depois, mesmo que o bot seja reiniciado.

Exemplos de coisas que queremos guardar:

  • XP e nível dos usuários
  • Saldo de moedas de economia do servidor
  • Configurações do servidor (canais de log, cargos padrão, prefixos, etc.)
  • Histórico simples (quantos comandos alguém usou, por exemplo)

Hoje vamos ver como fazer isso com arquivos JSON, uma solução simples e perfeita para projetos pequenos e médios.

▸ 2. O que é JSON?

JSON significa JavaScript Object Notation. É um formato de texto usado para representar dados estruturados (chave → valor).

Características do JSON:

  • É um arquivo de texto normal (ex.: dados.json)
  • Fácil de ler por humanos e por bots
  • Suportado nativamente em JavaScript e fácil de usar em Python
  • Perfeito para armazenar listas, objetos, configurações e estatísticas
Exemplo simples de JSON com dados de usuários:

{
  "1234567890": { "xp": 120, "nivel": 3 },
  "9876543210": { "xp": 45, "nivel": 1 }
}

Cada chave pode ser o ID do usuário e o valor é um objeto com os dados desse usuário.

▸ 3. Que dados faz sentido salvar para o bot?

Antes de escrever qualquer linha de código, é importante decidir o que será armazenado e como. Alguns exemplos práticos:

a) Sistema de XP:

{
  "id_usuario_1": { "xp": 250, "nivel": 5 },
  "id_usuario_2": { "xp": 80, "nivel": 2 }
}
b) Sistema de economia:

{
  "id_usuario_1": { "coins": 1500 },
  "id_usuario_2": { "coins": 270 }
}
c) Configurações do servidor:

{
  "id_servidor": {
    "canal_logs": "1234567890",
    "canal_boas_vindas": "0987654321",
    "cargo_padrao": "1122334455"
  }
}

A ideia é sempre a mesma: chave → valor. E o valor pode ser um objeto com todas as informações que fizerem sentido.

▸ 4. Ciclo básico: ler → alterar → salvar

Independente se for Python ou JavaScript, o fluxo geral para usar JSON como mini banco é sempre:

  • 1. Ler o arquivo JSON para a memória (carregar os dados)
  • 2. Alterar os dados conforme o que aconteceu (ex.: +10 de XP)
  • 3. Salvar de volta no arquivo JSON
Exemplo de fluxo com XP (conceito):

Evento: usuário mandou uma mensagem
Ação:
  - Ler dados.json
  - Somar +5 XP ao usuário
  - Ver se atingiu XP suficiente para upar de nível
  - Salvar dados.json com os valores atualizados

Nas próximas aulas você vai ver isso implementado em código real, mas aqui o foco é entender a lógica do processo.

▸ 5. Limitações e cuidados ao usar JSON

JSON é ótimo para começar, mas tem algumas limitações importantes:

  • Não é ideal para bots gigantes com milhares de usuários ativos
  • Se várias partes do bot salvarem ao mesmo tempo, pode dar conflito
  • Se o arquivo corromper (queda de energia, por exemplo), você pode perder dados

Boas práticas:

  • Manter cópias de segurança (backup) do arquivo JSON
  • Evitar salvar o arquivo a cada micro-ação (salvar em momentos chave)
  • Começar com JSON e, se o projeto crescer muito, migrar para um banco real

É exatamente por isso que, na Aula 9, vamos falar de bancos mais robustos, como SQLite e MongoDB.

JSON entra como a peça que falta para que tudo isso lembre do que aconteceu:

  • Registrar XP de quem mais participou do chat
  • Guardar preferências de cada usuário (linguagem favorita, área do conhecimento, etc.)

Assim, o bot começa a se comportar como um sistema de verdade, com memória e estado próprio.
