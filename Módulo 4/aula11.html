AULA 11 • Módulo 4 – Projeto Final

Criação de um Bot Completo (Projeto Final)

Objetivo da aula: unir tudo o que aprendemos desde o Módulo 1 para montar um bot completo, funcional, organizado, modularizado, com banco de dados, APIs externas, comandos, eventos, logs, sistema de XP e dashboard próprio.

▸ 1. Arquitetura geral do projeto

Antes de começar a programar, definimos como o projeto será organizado. Um bot completo precisa ter:

  • Módulo de Comandos (slash commands)
  • Módulo de Eventos (join, leave, mensagens, erros)
  • Sistema de Logs (console + canal logs)
  • Banco de Dados (JSON, SQLite ou MongoDB)
  • Sistema de XP
  • Comandos com API externa
  • Configurações por servidor
  • Dashboard Web (opcional)

Exemplo de estrutura de pastas:

/bot
├── config.json
├── index.js (ou main.py)
├── /comandos
│ ├── utilidade
│ ├── economia
│ └── geral
├── /eventos
│ ├── ready.js
│ ├── guildMemberAdd.js
│ └── interactionCreate.js
├── /sistema
│ ├── xp.js
│ ├── economia.js
│ └── logs.js
├── /database
│ ├── dados.json
│ ├── sqlite.db
│ └── mongodb.js
└── /web (dashboard)
└── index.html

Assim o bot fica organizado, modular e escalável.

▸ 2. Sistema completo de comandos (/)

Um bot completo usa Slash Commands organizados por categorias.

Comandos essenciais do projeto final:

  • /help – lista todos os comandos
  • /ping – status e latência
  • /clima – API externa
  • /serverinfo – dados do servidor
  • /perfil – estatísticas do usuário
  • /rank – ranking do XP

Cada comando deve ser um arquivo separado, limpo e modular.

▸ 3. Eventos completos

Eventos que o bot final deve implementar:

  • ready – quando o bot liga
  • guildMemberAdd – boas-vindas + logs
  • guildMemberRemove – saída + logs
  • interactionCreate – execução de comandos
  • messageCreate – sistema de XP
  • error – erros no bot

O event handler deve ser separado para manter o projeto limpo.

▸ 4. Sistema completo de XP e níveis

Com base no que aprendemos no Módulo 3, o bot final terá:

  • XP ao enviar mensagens
  • Cooldown por usuário para evitar spam
  • Curva de XP progressiva
  • /perfil → mostra XP, nível e progresso
  • /rank → ranking global
Exemplo de cálculo de nível:

xpNecessario = nivel * 100 nível = Math.floor(xpTotal / 100)

▸ 5. Banco de dados real integrado

O bot final deve permitir escolher:

  • JSON → versão simples
  • SQLite → versão intermediária
  • MongoDB → versão avançada (produção)

Estrutura mínima necessária no banco:

  • Tabela/coleção de usuários: xp, nível, coins, comandos usados
  • Tabela/coleção de servidores: canais, cargos, logs
  • Logs de comandos (opcional)

▸ 6. Integração com pelo menos 1 API externa real

Para o projeto final, o aluno precisa implementar pelo menos um comando que consome uma API externa real:

  • API de clima (OpenWeather)
  • API de CEP (ViaCEP)
  • API de moedas (AwesomeAPI)
Fluxo:

Usuário → /clima São Paulo Bot → Faz requisição na API Bot → Retorna embed estilizada com temperatura, clima, etc.

▸ 7. Conectar com o Dashboard (Aula 10)

O bot completo deve mandar dados para o dashboard ou permitir que o dashboard leia os dados dele pelo JSON/DB.

Informações recomendadas:

  • Status do bot
  • Uptime
  • Comandos mais usados
  • Total de servidores
  • Ranking de XP

▸ 8. Checklist completo do Projeto Final

Para considerar o bot realmente “completo”, ele precisa de:

  • ✔ Sistema de eventos
  • ✔ Slash Commands organizados
  • ✔ Sistema de XP
  • ✔ Logs completos
  • ✔ Banco de dados funcionando
  • ✔ API externa integrada
  • ✔ Dashboard básico

▸ 9. Resumo da Aula

  • Definimos a arquitetura final do projeto
  • Criamos o sistema completo de comandos e eventos
  • Consolidamos o sistema de XP e níveis
  • Integramos com APIs externas
  • Conectamos com banco de dados profissional
  • Planejamos um dashboard simples e funcional
  • Montamos o checklist final do projeto

Após esta aula, o aluno já tem material suficiente para construir um bot profissional do zero, organizado, modular e pronto para ser hospedado.
